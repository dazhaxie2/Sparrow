# 代理沟通约定

本文件是代理对外沟通语言的权威出处。`AGENTS.md` 的 Communication 节指向这里。

## 核心规则

**代理对用户的自然语言回复一律用中文。** 包括但不限于:

- 解释、总结、进度汇报、风险提示
- 计划、方案、选项说明
- 验证结果、命令输出解读
- 提问与澄清(用 AskUserQuestion 等工具时的选项文案)

## 不在约束范围内

下列内容保持各自既有的工程惯例,**不**因本规则改写为中文:

- 代码标识符:类名、方法名、变量名、函数名(JAVA/TS 惯例,英文)
- 代码注释:沿用周围代码既有语言(中/英/拼音以文件内一致性为准)
- commit message、PR 标题与描述:沿用仓库既有风格
- 命令、路径、标识符、报错原文:保持原样(英文/技术字符串),仅在其外层的解释用中文
- 现有 `docs/harness/` 与 `AGENTS.md` 文档正文:不因本规则回译

## 边界判断

- **引用技术内容时**:命令、变量名、文件路径、异常类名、HTTP 状态码等保持原样,用反引号包裹,外层解释用中文。
  - 例:`mvn -pl sparrow-ai -am test` 通过(✓);`运行 mvn 减 pl 测试`(✗,破坏可读性)。
- **混合输出时**:报告里夹杂数字、英文术语(如 SSE、Feign、Pinia)是自然的,不强求翻译成熟术语;整体句式与解释用中文。
- **工具输入**:写代码、写注释、写 commit、写文档内容时,按上一节惯例,不受"中文回复"约束。

## 为什么不能只靠对话提醒

每次会话都从 `AGENTS.md` 起步。把约定固化进 harness(本文件 + `manifest.json` 的 `requiredDocuments`)后:

1. `AGENTS.md` 首读即见,无需用户重复声明。
2. `node tools/harness.mjs guard` 的 H004 规则会校验本文件存在——删除或漏建会直接让守卫失败,形成可执行的反馈闭环。
