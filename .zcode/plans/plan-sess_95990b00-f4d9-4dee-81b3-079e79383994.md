## 优化背景

当前正在进行的主题是「加强 AI 聊天历史的健壮性」（区分加载失败/确实为空 + 重试）。沿此主题，覆盖你确认的三项：

---

## 1. 登录/登出同步历史（修复真实 bug）

**根因**：`AiChatPanel` 被 `<keep-alive>` 包裹，`onMounted` 只在首次挂载触发一次 → 换号登录后历史不会刷新；`logout()` 不清 chat store → 退出后仍残留上个账号的会话列表。

**改动**：
- `user/store.ts` → `logout()`：登出时调用 `useChatStore()` 清空 `sessions` / `activeSessionId` / `sessionsError`（避免跨账号历史残留）。在 chat store 里新增 `reset()` 方法集中清状态，logout 调它。
- `AiChatPanel.vue` → 把 `onMounted(() => if logged loadSessions)` 改为 `watch(() => user.isLoggedIn(), ...)`：登录态从 false→true 时拉历史，从 true→false 时清本地消息回欢迎页。覆盖"重新登录刷新"和"登出清屏"两个场景。

## 2. 修正过时注释

- `AiChatPanel.vue:61` 注释"未登录 loadSessions 内部会静默清空"与当前实现矛盾（store 已改为失败不清空）。改为准确描述：登录后预加载历史，列表加载由登录态 watch 驱动。

## 3. 删除/切换会话失败反馈

沿用 AI 模块**已有的内联反馈模式**（项目无全局 toast，`useToast` 零引用），不引入新机制：

- **删除失败**：`ChatHistoryDrawer.vue` 的 `confirmDelete()` 捕获 `removeSession` 返回 false，用组件内局部 ref 显示一行错误提示（复用现有 `.error-state` 样式风格），3 秒后自动消失。给"删除"按钮加短暂的 disabled 防重复点击。
- **切换会话失败**：`useChat.loadHistory()` 加载失败时不再静默回欢迎页，而是保留欢迎页 + push 一条 `mode:'error'` 的 assistant 消息提示"加载该对话失败，请重试"（与未登录提示同一渲染路径，无需新样式）。
